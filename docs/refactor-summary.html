根因 & 技术选型

「样式不统一」到底是什么造成的

根因：split-run 样式塌缩。 旧实现用手写正则改 OOXML <w:t> 节点做变量替换。Word/输入法常把一个 {{客户名称}} 占位符拆成多个样式不同的 run（如 {{ / 客户名称(加粗) / }}），手写替换把整个值塞进第一个 run 的样式里，变量真正的样式（加粗等）被丢弃 —— 于是「导入后样式和最终效果不一致」。

选型（站在巨人肩膀上）： 实测对比后选 easy-template-x（MIT，纯 JS，字面 key 替换，支持任意字段名、多行自动换行、图片插件）。docx-templates 因把 {{}} 当 JS 表达式、对含空格/冒号/连字符的字段名报错而弃用。替换前先做 run 归一化（把占位符涉及的相邻 run 样式统一为「变量名重叠最多的 run」），样式保真。Docx 与飞书两条路径同思路。
